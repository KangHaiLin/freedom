# 用户场景映射表模板

## 概述

本模板用于将用户场景映射到具体需求，验证需求对关键业务场景的支持完整性。通过系统化的场景映射，确保每个重要用户场景都有相应的需求支持。

## 使用说明

### 映射目的
1. **完整性验证**：验证需求是否支持所有关键用户场景
2. **可追溯性建立**：建立从场景到需求的追溯关系
3. **场景覆盖评估**：评估需求对场景的覆盖程度
4. **缺口识别**：识别场景中缺失的需求支持

### 映射方法
1. **选择关键场景**：从《使用场景文档》中选择关键场景
2. **分解场景步骤**：将场景分解为具体步骤
3. **识别需求支持**：为每个步骤识别支持的需求
4. **评估支持质量**：评估需求对步骤的支持质量
5. **记录映射结果**：记录映射结果和发现的问题

### 质量评估标准
- **完全支持**：需求完全支持场景步骤
- **部分支持**：需求部分支持场景步骤，需要补充
- **不支持**：需求不支持场景步骤，需要新增需求
- **不适用**：步骤不需要需求支持（如人工操作）

## 关键场景映射示例

### 示例1：量化研究员开发新策略（UC-01）
#### 场景描述
量化研究员需要开发、测试和部署新的交易策略，包括数据获取、策略编写、回测、优化和部署。

#### 关键步骤映射摘要
| 步骤 | 涉及需求 | 支持程度 | 主要问题 | 优先级 |
|------|----------|----------|----------|--------|
| 1. 获取历史数据 | REQ-DM-001, REQ-DM-002 | 完全支持 | 无 | - |
| 2. 编写策略代码 | REQ-SR-001, REQ-SR-003 | 部分支持 | 缺少代码模板和示例 | P1 |
| 3. 执行回测 | REQ-SR-002 | 处理中 | 回测速度指标需验证 | P1 |
| 4. 分析回测结果 | REQ-SR-004 | 不支持 | 缺少可视化分析需求 | P2 |
| 5. 策略优化 | REQ-SR-005 | 部分支持 | 优化算法支持不明确 | P1 |
| 6. 策略部署 | REQ-TE-001, REQ-TE-002 | 完全支持 | 无 | - |

### 示例2：交易员执行算法交易（UC-02）
#### 场景描述
交易员需要监控市场、创建订单、执行算法交易、监控执行情况和风险管理。

#### 关键步骤映射摘要
| 步骤 | 涉及需求 | 支持程度 | 主要问题 | 优先级 |
|------|----------|----------|----------|--------|
| 1. 市场监控 | REQ-DM-001, REQ-DM-003 | 完全支持 | 无 | - |
| 2. 订单创建 | REQ-TE-001 | 已解决 | 订单确认机制已明确 | - |

## 使用建议

### 1. 评审团队分工
- **业务代表**：负责主要业务场景映射
- **用户体验专家**：负责体验相关评估
- **需求分析师**：负责需求匹配度评估
- **测试代表**：负责可测试性评估

### 2. 评审流程
1. **场景选择**：选择关键场景进行映射（建议3-5个）
2. **步骤分解**：详细分解场景步骤
3. **需求匹配**：匹配每个步骤的需求支持
4. **评估记录**：记录评估结果和问题
5. **汇总分析**：汇总多个场景的共性问题
6. **改进建议**：提出具体的改进建议

### 3. 输出物要求
- 完整的场景映射表（每个场景一份）
- 综合评估报告
- 改进建议清单
- 问题跟踪表更新

### 4. 质量检查点
- [ ] 所有关键场景都已评审
- [ ] 每个步骤都有明确的需求匹配
- [ ] 支持程度评估客观准确
- [ ] 问题描述具体清晰
- [ ] 改进建议具体可行
- [ ] 优先级设置合理
