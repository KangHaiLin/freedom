## 3. 测试场景设计

### 3.0 场景0：数据预热性能对比测试（冷启动 vs 热缓存）

#### 场景描述
对比冷启动（首次加载）和热缓存（数据已在内存中）两种状态下的回测性能，评估数据预热对总体性能的影响，为生产环境部署提供优化依据。

#### 测试条件
| 参数 | 数值 | 说明 |
|------|------|------|
| **策略类型** | 双均线策略（MA20/MA60交叉） | 与场景2保持一致，便于对比 |
| **数据规模** | 10年日线数据（62.5百万数据点） | 全市场5000只股票 |
| **测试模式** | 冷启动模式 + 热缓存模式 | 冷启动：重启进程后首次运行；热缓存：同一进程内重复运行 |
| **测试次数** | 冷启动：3次独立进程运行；热缓存：5次连续运行 | 取平均值，对比性能差异 |
| **数据加载机制** | Parquet格式 + 内存映射 | 优化数据加载性能 |

#### 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **冷启动回测时间** | 基准测量值 | 进程重启后首次运行的总时间 |
| **热缓存回测时间** | 基准测量值 | 同一进程内重复运行的总时间 |
| **预热性能提升** | ≥30% | (冷启动时间 - 热缓存时间) / 冷启动时间 |
| **数据加载时间差异** | 基准测量值 | 冷启动 vs 热缓存数据加载时间 |
| **内存缓存效果** | 内存使用稳定 | 热缓存模式下内存使用应稳定 |

#### 测试目的
1. 量化数据预热对回测性能的实际影响
2. 评估生产环境是否需要预热机制
3. 为系统部署策略提供数据支持

#### 测试条件
| 参数 | 数值 | 说明 |
|------|------|------|

| **并发用户数** | 1个（单进程） | 基础性能测试 |
| **测试次数** | 5次 | 取平均值，排除异常值 |
| **预热轮次** | 1次 | 预热Python JIT和文件缓存 |

#### 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **回测执行时间** | ≤60秒（基础目标） | 从策略初始化到结果输出的总时间 |
| **数据加载时间** | ≤10秒 | 数据从磁盘加载到内存的时间 |

### 5.2 数据分析方法

#### 性能指标计算
```python
# 回测性能分析脚本框架
import pandas as pd
import numpy as np
from scipy import stats

def analyze_backtest_performance(test_results):
    """分析回测性能测试结果"""

    # 基本统计量
    analysis = {
        'scenario': test_results['scenario_name'],
        'sample_size': len(test_results['execution_times']),
        'mean_time': np.mean(test_results['execution_times']),
        'std_time': np.std(test_results['execution_times']),
        'min_time': np.min(test_results['execution_times']),
        'max_time': np.max(test_results['execution_times']),
        'p50_time': np.percentile(test_results['execution_times'], 50),
        'p95_time': np.percentile(test_results['execution_times'], 95),
        'p99_time': np.percentile(test_results['execution_times'], 99),
    }

    # 与目标值对比
    target_time = test_results['target_time']
    analysis['target_achievement'] = analysis['mean_time'] <= target_time
    analysis['deviation_percent'] = (analysis['mean_time'] - target_time) / target_time * 100

    # 资源使用分析
    analysis['peak_memory_mb'] = max(test_results['memory_usage']) / 1024 / 1024
    analysis['avg_cpu_percent'] = np.mean(test_results['cpu_usage'])

    return analysis
```
