# 子系统详细设计

## 1. 数据管理子系统 (DM)
### 1.1 模块划分
```
┌─────────────────────────────────────────────────────┐
│                  数据管理子系统                      │
├───────────────┬───────────────┬─────────────────────┤
│ 数据采集模块  │ 数据清洗模块  │ 数据存储模块         │
├───────────────┼───────────────┼─────────────────────┤
│ 数据分发模块  │ 数据质量模块  │ 元数据管理模块       │
└───────────────┴───────────────┴─────────────────────┘
```

### 1.2 核心功能
- **数据采集**：支持Wind、Tushare、JoinQuant等多数据源接入，支持实时行情、历史行情、财务数据、公告数据等采集
- **数据清洗**：缺失值填充、异常值检测、数据格式标准化、重复数据删除
- **数据存储**：根据数据类型和使用场景自动路由到合适的数据库
- **数据分发**：通过WebSocket、消息队列、REST API等方式提供数据服务
- **数据质量**：数据完整性校验、一致性校验、时效性监控，质量告警
- **元数据管理**：数据字典管理、数据血缘追踪、数据生命周期管理

### 1.3 核心流程
#### 实时数据采集流程
1. 定时从数据源拉取最新行情数据 → 2. 数据格式校验和清洗 → 3. 写入Kafka消息队列 → 4. 实时计算模块处理 → 5. 写入内存数据库供实时查询 → 6. 异步写入时序数据库持久化

#### 历史数据回测读取流程
1. 接收回测数据查询请求 → 2. 权限校验和参数验证 → 3. 查询ClickHouse获取批量历史数据 → 4. 数据格式转换 → 5. 批量返回给回测引擎 → 6. 记录查询日志和性能指标

## 2. 策略研究子系统 (STR)
### 2.1 模块划分
```
┌─────────────────────────────────────────────────────┐
│                  策略研究子系统                      │
├───────────────┬───────────────┬─────────────────────┤
│ 策略开发模块  │ 回测引擎模块  │ 策略优化模块         │
├───────────────┼───────────────┼─────────────────────┤
│ 绩效分析模块  │ 模拟交易模块  │ 策略库管理模块       │
└───────────────┴───────────────┴─────────────────────┘
```

### 2.2 核心功能
- **策略开发**：支持Python语言策略编写，提供丰富的因子库、指标库、工具函数
- **回测引擎**：支持日线、分钟线、Tick级数据回测，支持T+1、涨跌停等A股规则
- **策略优化**：支持参数寻优、遗传算法、网格搜索等优化方法
- **绩效分析**：提供收益率、最大回撤、夏普比率、胜率等完整绩效指标
- **模拟交易**：支持实时行情模拟交易，验证策略实盘表现
- **策略库管理**：策略版本控制、权限管理、上线审批流程

### 2.3 核心流程
#### 回测执行流程
1. 用户提交回测任务 → 2. 任务调度分配资源 → 3. 加载策略代码和参数 → 4. 读取历史行情数据 → 5. 逐K线执行策略逻辑 → 6. 记录交易信号和持仓变化 → 7. 计算绩效指标 → 8. 生成回测报告 → 9. 通知用户回测完成

## 3. 交易引擎子系统 (TS)
### 3.1 模块划分
```
┌─────────────────────────────────────────────────────┐
│                  交易引擎子系统                      │
├───────────────┬───────────────┬─────────────────────┤
│ 订单管理模块  │ 算法交易模块  │ 执行路由模块         │
├───────────────┼───────────────┼─────────────────────┤
│ 持仓管理模块  │ 仓位计算模块  │ 交易对账模块         │
└───────────────┴───────────────┴─────────────────────┘
```

### 3.2 核心功能
- **订单管理**：订单创建、修改、撤销、状态跟踪，支持多种订单类型
- **算法交易**：支持VWAP、TWAP、冰山订单、智能路由等算法交易策略
- **执行路由**：根据订单特点智能选择最优交易通道和券商接口
- **熔断保护**：系统过载或行情异常时自动熔断，防止雪崩效应，支持服务降级
- **持仓管理**：实时持仓计算、成本核算、收益跟踪
- **仓位计算**：根据风险指标和策略信号自动计算下单仓位
- **交易对账**：日终与券商交割单对账，确保交易数据一致性

### 3.3 熔断机制设计
#### 熔断触发条件
1. 系统CPU使用率超过85%持续5分钟
2. 订单处理延迟超过100ms持续1分钟
3. 券商接口成功率低于90%持续1分钟
4. 行情数据延迟超过5秒持续30秒

#### 熔断策略
| 熔断级别 | 策略 |
|----------|------|
| 一级熔断 | 拒绝新的回测任务请求，保障交易核心功能 |
| 二级熔断 | 暂停算法交易，仅支持手工下单 |
| 三级熔断 | 自动撤掉所有未成交订单，禁止新开仓，仅允许平仓 |
| 四级熔断 | 暂停所有交易功能，进入系统保护状态 |

#### 熔断恢复
- 系统指标恢复正常后，自动逐级恢复功能
- 人工确认后可强制恢复所有功能
- 熔断事件全程记录审计日志

### 3.3 核心流程
#### 实盘交易流程
1. 策略生成交易信号 → 2. 风控系统前置检查 → 3. 订单创建和参数验证 → 4. 算法交易模块拆单 → 5. 执行路由选择最优通道 → 6. 报单到券商接口 → 7. 实时跟踪订单状态 → 8. 成交回报处理 → 9. 更新持仓和资金 → 10. 记录交易日志

## 4. 风险管理子系统 (RM)
### 4.1 模块划分
```
┌─────────────────────────────────────────────────────┐
│                  风险管理子系统                      │
├───────────────┬───────────────┬─────────────────────┤
│ 实时风控模块  │ 合规检查模块  │ 头寸管理模块         │
├───────────────┼───────────────┼─────────────────────┤
│ 压力测试模块  │ 风险计量模块  │ 风险报表模块         │
└───────────────┴───────────────┴─────────────────────┘
```

### 4.2 核心功能
- **规则引擎**：可视化合规规则配置，动态生效无需重启系统
- **实时风控**：交易前风险检查、盘中实时监控、阈值告警
- **合规检查**：持仓比例限制、交易频率限制、异常交易检测、合规报表生成
- **头寸管理**：单一标的持仓限制、行业持仓限制、总仓位限制
- **压力测试**：历史情景模拟、极端行情测试、风险价值(VaR)计算
- **风险计量**：波动率、最大回撤、风险敞口、流动性风险等指标计算
- **风险报表**：日报、周报、月报生成，监管报送材料自动生成

### 4.3 规则引擎设计
#### 架构
```
┌─────────────────────────────────────────────────────┐
│ 规则管理控制台：可视化配置、版本管理、测试验证      │
├─────────────────────────────────────────────────────┤
│ 规则引擎核心：Drools规则引擎、执行器、事实对象      │
├─────────────────────────────────────────────────────┤
│ 规则存储：规则库、版本历史、生效配置                │
└─────────────────────────────────────────────────────┘
```

#### 核心能力
- 可视化规则配置，业务人员无需编码即可配置合规规则
- 规则版本管理，支持灰度发布、回滚
- 规则测试验证，上线前可模拟测试效果
- 高性能：单条规则执行时间<1ms，支持每秒10000次规则校验
- 动态生效：规则修改后无需重启系统，实时生效

#### 内置规则模板
- 持仓限制类：单票持仓比例、行业持仓比例、总仓位限制
- 交易限制类：涨跌停限制、T+1交易限制、单日交易次数限制
- 异常交易类：频繁报撤单、对倒交易、大额交易监控
- 风控指标类：最大回撤限制、止损限制、风险敞口限制

### 4.4 合规审计报表自动生成设计
#### 报表类型
| 报表名称 | 生成频率 | 报送对象 |
|----------|----------|----------|
| 交易日报 | 每日 | 内部风控、运营 |
| 风险周报 | 每周 | 管理层、风控委员会 |
| 合规月报 | 每月 | 监管部门、公司管理层 |
| 异常交易报告 | 实时触发 | 风控部门、监管部门 |
| 年度审计报告 | 每年 | 审计部门、监管部门 |

#### 生成流程
1. 数据采集：自动从交易系统、风控系统、账户系统采集所需数据
2. 数据校验：校验数据完整性、一致性，异常数据告警
3. 报表生成：根据监管要求模板自动生成报表内容
4. 审核流程：支持多级审核，审核通过后自动报送
5. 归档存储：报表永久归档存储，支持随时查阅导出


### 4.3 核心流程
#### 交易前风控流程
1. 订单进入风控系统 → 2. 用户权限校验 → 3. 资金可用性检查 → 4. 持仓限制检查 → 5. 涨跌停限制检查 → 6. 异常交易检测 → 7. 风险敞口计算 → 8. 通过/拒绝订单 → 9. 记录风控日志

## 6. 系统管理子系统 (SYS)
### 6.1 模块划分
```
┌─────────────────────────────────────────────────────┐
│                  系统管理子系统                      │
├───────────────┬───────────────┬─────────────────────┤
│ 用户管理模块  │ 权限管理模块  │ 系统监控模块         │

```

### 6.2 核心功能
- **用户管理**：用户创建、修改、删除、角色分配
- **权限管理**：基于RBAC的权限控制，细粒度功能权限和数据权限
- **系统监控**：服务器监控、服务监控、数据库监控、性能监控
- **日志管理**：操作日志、系统日志、交易日志的查询和审计
- **配置管理**：系统配置、参数配置、策略配置的统一管理
- **运维工具**：一键部署、备份恢复、故障排查工具
