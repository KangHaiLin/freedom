# 详细设计文档总览

## 概述
本文档是A股量化交易软件系统的详细设计说明书，是架构设计的细化和落地，作为开发人员编码实现的直接依据。

## 设计范围
详细设计覆盖所有核心子系统，分阶段完成：
1. **第一阶段（第1迭代）**：数据管理子系统详细设计
2. **第二阶段（第2迭代）**：策略研究子系统、交易引擎子系统详细设计
3. **第三阶段（第3迭代）**：风险管理子系统、用户交互子系统、系统管理子系统详细设计

## 设计规范
### 命名规范
- 类名：大驼峰命名法（如：`MarketDataIngestor`）
- 方法名：小驼峰命名法（如：`getRealtimeQuote`）
- 变量名：小驼峰命名法（如：`stockCode`）
- 常量名：全大写+下划线（如：`MAX_RETRY_TIMES`）
- 数据库表名：小写+下划线（如：`stock_daily`）
- 接口路径：小写+短横线（如：`/api/v1/market/realtime`）

### 文档规范
每个子系统详细设计包含以下内容：
1. 模块划分和职责说明
2. 核心类设计（类图、属性、方法）
3. 接口详细设计（参数、返回值、错误码）
4. 业务流程设计（时序图、流程图）
5. 数据库表结构详细设计（字段、索引、约束）
6. 异常处理设计
7. 单元测试用例要点
