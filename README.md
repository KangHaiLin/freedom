# A股量化交易软件项目

## 项目概述

本项目旨在开发一个全功能的A股量化交易软件平台，服务于机构交易员、零售投资者和量化研究员。项目采用敏捷开发方法，分阶段实施。

## 当前状态

**阶段1：需求收集与框架建立** - 已完成
- 创建了基于IEEE 830标准的软件需求规格书（SRS）框架
- 完成了利益相关者分析和使用场景文档
- 整理了A股市场规则摘要
- 建立了Git版本控制

## 文档说明

### 1. 软件需求规格书（SRS）
基于IEEE 830标准，包含：
- **第1部分：引言** - 目的、范围、定义、参考文献
- **第2部分：总体描述** - 产品视角、功能、用户特征、约束
- **第3部分：具体需求** - 功能需求、非功能需求、接口需求等

### 2. 利益相关者分析
识别和分析所有项目利益相关者，包括：
- 量化研究员、交易员、风险经理、合规官
- 系统管理员、开发团队、测试团队
- 监管机构、数据供应商、券商合作伙伴

### 3. 使用场景
描述典型用户场景，包括：
- 量化研究员开发新策略
- 交易员执行算法交易
- 风险经理监控系统风险
- 系统管理员日常运维
- 合规官进行合规检查

### 4. A股规则摘要
汇总中国A股市场主要规则：
- 交易时间和制度
- 涨跌幅限制和交易费用
- 市场监管和合规要求
- 数据接口和风险管理

## 技术栈

- **主要语言**：Python 3.8+
- **数据存储**：PostgreSQL（关系型）、InfluxDB/ClickHouse（时序数据）
- **消息队列**：RabbitMQ/Kafka
- **前端框架**：React/Vue.js
- **部署环境**：Linux服务器，Docker容器化
